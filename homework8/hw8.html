Explanation of AJAX

Web applications can update and retrieve data from a server asynchronously without reloading the entire page thanks to a technique called Asynchronous JavaScript and XML, or AJAX. AJAX is a collection of technologies rather than a programming language.

This code uses AJAX to retrieve and show XML data in an HTML table. The cd.xml file is loaded asynchronously by the loadDoc() function. It sends a GET request using the XMLHttpRequest object. The myFunction() function receives the response data from the onload event when the request is finished.

The responseXML property in myFunction() is used to parse the XML response into an XMLDocument object. The XML document's "CD" elements are then all retrieved by the code. Using a loop, it iterates through each "CD" element, extracting the text content of the "ARTIST" and "TITLE" child elements. It creates an HTML table beginning with a header row that contains "Artist" and "Title." In the table, these values are added as rows.

Rest API

A REST API (Representational State Transfer Application Programming Interface) is a way to interact with a system or service over HTTP. REST APIs follow a set of principles that make them lightweight, scalable, and easy to use. When you choose GitHub’s REST API, it allows you to programmatically interact with GitHub repositories, users, issues, pull requests, and more.

The GitHub REST API primarily returns data in JSON (JavaScript Object Notation) format

Do You Need a Key to Access the API?

  • Yes, certain endpoints require authentication to access. GitHub provides several methods for authentication:
    1. Personal Access Tokens (PAT):
    2. OAuth Tokens: Used for third-party integrations.
    3. GitHub Apps Authentication: For specific apps installed in GitHub organizations or user accounts.
  • Some endpoints (like public user data) do not require authentication, but authenticated requests allow:
    • Higher rate limits.
    • Access to private repositories or user-specific data.

Do Companies Need to Pay Any Fee to Use the API?

  • Free for Basic Use: GitHub’s REST API is free for most use cases, especially for personal and open-source projects.
  • Paid Features: Companies need to pay for GitHub Enterprise or GitHub Team plans to:
    1. Access private repositories.
    2. Use advanced features like Actions (CI/CD) or advanced security.
    3. Have higher limits on API calls and enterprise-level support.
  • Charges for Rate Limit Increases:
    1. Free-tier: 60 requests per hour (unauthenticated), 5000 requests per hour (authenticated).
    2. Higher rate limits may be available for enterprise plans.

Summary

  • Response Format: JSON.
  • Authentication: Required for most actions; tokens are used.
  • Fees: No fees for basic use; enterprise or team plans involve payment for private repos, advanced features, and rate-limit increases.
